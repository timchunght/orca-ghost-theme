Orca Theme

Orca is a minimal, yet feature full theme for the Ghost blogging platform. Ideal for personal web blogs, magazines and web application blogs. Equipped with the latest web technologies such as responsive Layout, Orca looks perfect on all devices and resolutions, from Mobile to the latest 4K monitors. Orca extends upon the ordinal functionality of the Ghost platform, with a range of 3 post styles, a covered feature post page, extended tag support, multi depth navigation options and post comments. Supports the latest features added in the latest Ghost 0.4.2 release and includes compatibility support for future versions.

  • Fully Responsive

    Optimized for Mobile, Tablets, Notebooks, and Desktop Devices. Developed on iPhone, iPad, and Desktop.

  • High Resolution Support

    CSS3 elements & SVG icons scale perfectly on all resolutions, including high resolution devices such mobile phones, retina displays, and 4K monitors.

  • Custom Post Types

    Orca comes included with three custom post styles, easily edit the look of your post titles by adding a unique tag.

  • Page Layouts

    Easily switch between left aligned sidebar, or right aligned sidebar without needing to edit any code.

Self Hosted Install

  • 1

    Locate the theme directory within the download files, it can be found at theme/Orcatheme.

  • 2

    Login to FTP/SFTP of your blog, or the file browser of your provided hosting service.

  • 3

    Navigate to the /content/theme directory within your ghost installation.

  • 4

    Upload the Orcatheme directory found in step one into the remote /content/theme directory.

  • 5

    Restart your Ghost service. The process for this will change dependent on your provider. If you are unaware how to restart the service please raise a support issue with your host.

  • 6

    Login to your Ghost admin panel. This can be found at yourdomain.com/ghost.

  • 7

    Navigate to the Settings > General tab.

  • 8

    In the Theme drop down, select the newly installed Orca Theme. Click Save to save the theme changes.

Ghost.io Install

  • 1

    Login to your Ghost.io user panel and navigation to the My Blogs page.

  • 2

    Select the Settings button on the blog you wish to upload the theme.

  • 3

    On the Custom Theme option, click the Choose File button and navigate to the orcatheme.zip file in the downloaded theme directory.

  • 3

    Upload the theme and Save Settings. You may have to activate the theme in your ghost admin panel.

Customization

Blog Options

  • Featured Post Images
    Each featured post can have a feature image which is displayed within the header of the post page. More information on this feature and how to use it can be found below in the Features section.
  • Post Types
    Orca comes included with 3 different post styles, which can be seen on the post list and on the individual post pages. For more information on how to use these post styles can be seen in the Post Types section below.
  • Posts Per Page
    For best theme visuals we recommend setting this to any odd number above 5.
  • Blog Image
    Upload your blog logo here, it will be displayed on a white background so should be made to contrast well with this color. This logo is displayed in the sidebar. A solid black logo is recommended.
  • Blog Cover Image
    Upload your blog logo here, this version however will be shown on featured cover post pages, and will need to contrast with a dark background. A solid white logo is recommended.

Theme Options

These options can be found within the file /assets/js/global.min.js with included documentation. These are all located on the first line of the file. Replace the current values with your required settings.

  • Disqus Shortname
    Enter your Disqus account shortname here to enable Disqus comments for blog posts. Requires an account, which can be obtained for free at http://disqus.com. More information can be found below.
  • Google Comments
    If true, and a Disqus shortname is not supplied (see above), this option will use Google+ Comments within your blog posts. More information can be found about Google Comments in the Features -> Google Comments section.
  • Google Analytics ID
    Enter your Google Analytics ID here to enable analytic service. Requires an account, which can be obtained for free at http://analytics.google.com. More information can be found below.
  • Google Analytics Domain
    The registered URL for the Google Analytics account should be entered here.
  • Enable Syntax Highlighting
    If true, will enable syntax highlighting of code blocks automatically within posts. Disable by settings value to false.
  • Enable Auto-Load of Post Comments
    If true, will auto load comments section on page/post load.
If you misconfigure you can replace the /assets/js/global.min.js with the original file to restore defaults.

Post Types

Orca has 3 different post styles (excluding the featured post layout) which allows you adjust how your post appears within the post list. These 3 post styles are enabled using the post tags within the Ghost editor. If a post type tag is not set the default post style will be used using a plain text header and no image. These tags are hidden and will not appear on the post page. The Feature and Banner post styles both use a post image, which has to be set within the post editor by placing an image with a specific alt tag between your first and second paragraph. E.g.

This is my first paragraph of text.

![postimage]

This is my second paragraph of text.

Previews of each post type can be seen on the live preview here. Hover of the navigation icon, and follow to the Post Types drop down.

Feature

This style will include a large image at the top of the post, with the post header within it. Two paragraphs of the post will be shown on post list pages. To enable this post style add the tag 'feature' as the last post tag. (Note this is not the same as a Featured post)

Banner

This style will include a small banner image at the top of the post, with the post header below it. To enable this post style add the tag 'feature' as the last post tag.

Standard (Default)

If neither of the above post styles are used, the post will default two the standard post styles, which incudes the post title and first paragraph of text.

Feature Configuration & Use

Featured Posts

Featured blog posts have an adjusted header, which displays a large cover image and the title of the post. This image can be customized by the user, or left blank for the default dark background.

Creating a featured post is simple, first create your standard blog post and add your content. Within the post markdown editor add the following code between the first and second paragraph ![postimage]. This will create an image upload area in the live preview section of the editor. Upload the image you wish to use as the featured background. For improved performance its recommended that any large images are reduced in size (1680 width is recommended) and compressed. Below is an example of how the code might look in your editor.

This is my first paragraph of text.

![postimage]

This is my second paragraph of text.

Once you have added content to your post, and a feature image, publish the post. Return back to the posts index, and click the star on the submitted blog post. The post is now featured and will contain a full page header. Previous posts can by also be featured by clicking the star icon, and adding a post image.

Social Network Profiles

Orca includes support for linking to social platforms such as facebook, twitter, github etc. within the sidebar and the footer of each page. Configure your social links by editing the file Orcatheme/partials/social_sidebar.hbs for the sidebar and Orcatheme/partials/social_footer.hbs for the footer.

Edit each lines href="" value, including your profile link for that particular platform. Delete any lines for platforms which are not wanted. They can be added again later at any time by copying them from the original source file. Remember to edit for the sidebar and footer links.

Static Pages & Navigation

Ghost release 0.4.0 added support for static pages which can be added to your blogs navigation. In order to create a static page create a new blog post with the pages content and header. Once created click the post settings icon and tick to enable the Static Page option. Copy the post URL which is shown in the same window. This URL needs to be added to the navigation section of the theme files. Open up Orcatheme/partials/navigation.hbs and edit the navigation sections to include the link to your page and a desired name. Below is an example of the navigation section. Reference Lines 30 & 44.

{{! ADD STATIC PAGE LINKS HERE }}
{{! REMOVE THE THREE DEFAULT LINKS BELOW IF NOT USED }}
{{! REPEAT BY ADDING LINKS TO THE FOOTER BELOW AS COMMENTED}}
<ul class="menu">
	<li><a href="/about/">About</a></li>
	<li>
		<a href="#">Features <i class="fa fa-chevron-down"></i></a>
		{{! BELOW IS A NESTED MENU }}
		<ul>
			<li><a href="#">Nested Option One</a></li>
			<li><a href="#">Nested Option Two</a></li>
			<li><a href="#">Nested Option Three</a></li>
		</ul>
	</li>
	<li><a href="/contact/">Contact</a></li>
</ul>

Disqus Comments

Setting up Discus comments for individual posts is simple. Firstly you will need an Disqus account, which can be obtained freely at http://disqus.com. Once you have created an account and setup your shortname, enter it into the theme configuration which can be found within the /assets/js/global.min.js file.

Once you have setup and enabled comments, all administration for comments can be done via the Disqus control panel. To access this admin panel navigate to Disqus, and select 'Admin' located in the header of the page. If at any time you wish to disable comments, you can do this by deselecting the 'Use Disqus Comments' theme option.

Google Comments

If you prefer, you can enable Google Plus comments to replace Disqus comments. To do this enable the setting in the configuration file located at /assets/js/global.min.js. If a Disqus shortname is set, this will override Google Comments, so set Disqus to blank ''. Please note, Google Plus Comments are unsupported by Google, and we recommend that Disqus is used instead. The implementation used to integrate Google Comments is not official or documented, and may stop working without notice at any time.

Tag Cloud

On the sidebar of each page is a tag cloud which needs to be manually edited with your most popular tags, or main categories. To do this open the orcatheme/partials/tags.hbs file, and observe the structure of the default links. To add a new tag copy and paste the line from an existing tag and edit the appropriate fields. Delete the lines of unwanted tags to remove them, such as the theme examples.

{{! ADD COMMON TAGS HERE FOR SIDEBAR }}
{{! REMOVE THE DEFAULT TAGS BELOW IF NOT USED }}
<ul class="tags_list">
	<li><a href="{{@blog.url}}/tag/taghere">taghere</a></li>
</ul>

Syntax Highligting

Orca comes with the Rainbow plugin preinstalled. This plugin will syntax highlight and format any code which you post in your blog. A code block can be created by tabing in a section of code twice in the markdown editor, or by 4 spaces. Too disable this feature edit the theme configuration file located at /assets/js/global.min.js.

// THIS IS AN EXAMPLE OF JS SYNTAX HIGHLIGHTING

Post Annotations

Within your blog posts you can include annotations for paragraphs of text, images, and code. To do this requires using some HTML markup inside the post editor. You can copy and paste the section of code below into your editor and replace the comments with the desired content. If you are annotating code sections, you may need to wrap the code in <code> and <pre> tags.

<div class="annotation">
ADD YOUR MAIN CONTENT HERE E.G. Image, Code.
<div class="main">
<h5>Annotation Title Here</h5>
<p>Annotation text here.</p>
</div>
</div>

Google Analytics

Orca supports Google Analytics on all pages. Enabling this feature requires an active account with the Google Analytics service (http://analytics.google.com). Once an account is created, and website registered, the UID code is required which is a similar format to the following: UA-28282324-1. Copy this code into the configuration file which can be found at assets/js/global.min.js.

Grunt (Developers)

This feature is intended for developers. Orca implements SASS stylesheets, which compile to CSS. Included with Orca is a Gruntfile which allows easy compiling of SASS into CSS, aswell as minifying all the source files into assets. Providing you have the prerequisites on your server you can install the dependencies using npm install. Compile the source using grunt. More information on Grunt and its setup can be found at http://gruntjs.com/.
